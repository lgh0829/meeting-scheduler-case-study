# 모두에게 괜찮은 시간, 어떻게 찾을까

> 6명이 모이는 1시간짜리 회의. 누군가는 점심 직후를 피하고 싶고, 누군가는 특정 요일에 외근이 많고, 꼭 와야 하는 사람과 빠져도 되는 사람이 섞여 있습니다. 이 조건들을 반영해 회의 시간을 빠르게 확정하는 경험을 설계했습니다.

## 어떤 문제인가

회의 일정을 잡을 때 6명의 조건을 모두 같은 무게로 다룰 수는 없습니다. 가장 중요한 차이는 참석의 무게입니다. 꼭 참석해야 하는 사람이 있고, 빠져도 괜찮은 사람이 있습니다.

그런데 기존 일정 조율 도구에는 이 차이를 담을 수 없는 한계가 있습니다. 응답자 전원을 동일하게 취급해 **"전원 가능한 시간"만 찾다** 보니, 빠져도 괜찮은 한 명의 제약이 필수 참석자 전원이 가능한 현실적인 시간까지 지워버립니다. 결국 주최자는 스케줄링 도구 대신 메신저 도구를 사용하여 개별 조율에 많은 시간을 쏟게 됩니다.

## 어떻게 풀었나

주최자로부터 참석의 무게를 스케줄링 도구가 알게 했습니다. 여기에 제약의 성격(업무 일정 때문에 불가능한 시간인지, 피하고 싶은 시간인지)을 한 축 더 얹어, 네 가지 조합을 각각 다르게 다룹니다.

| | 필수 참석자 | 선택 참석자 |
|---|---|---|
| **업무 일정** (외근, 다른 회의) | **하드 필터.** 겹치는 시간은 후보에서 제외합니다 | **1순위 신호.** 참석 가능한 인원이 많을수록 후보가 위로 올라갑니다 |
| **선호 시간** (점심 직후 회피 등) | **1순위 신호.** 회피 시간을 피한 후보가 위로 올라갑니다 | **2순위 신호.** 후보가 비슷할 때 참고합니다 |

- **선택 참석자의 무응답은 진행을 막지 않습니다.** 필수 참석자의 응답만 모이면 후보 탐색이 시작됩니다.
- **확정 통보도 무게에 맞춥니다.** 필수 참석자에게는 확정 사실을, 선택 참석자에게는 "참석이 어려우면 빠져도 괜찮다"는 메시지를 보냅니다.

후보 시간은 "전원 가능한 시간" 하나가 아니라 어떤 조건을 얼마나 만족하는지 비교할 수 있는 정보로 나타나고, 마지막 선택은 주최자가 내립니다.

## 왜 이 방식으로 설계했나

두 가지 대안을 검토하고 배제했습니다.

| 대안 | 내용 | 배제한 이유 |
|---|---|---|
| **A. 취합 자동화만 강화** | 흩어진 응답을 한곳에 모으는 데 집중 | 취합을 자동화해도 탐색 로직이 전원 만장일치를 요구하면 같은 문제가 반복됩니다. 취합은 이 설계의 전제 단계로 포함했습니다 |
| **B. 참석자별 가중치 점수화** | 1~5점 척도로 우선순위를 세분화 | 주최자에게 참석자마다 점수를 매기는 입력 부담을 더합니다. 필수/선택이라는 이분법이 이미 주어져 있어, 그보다 정교한 표현은 과한 설계라고 판단했습니다 |

결국 "필수 참석자의 조건은 반드시 지켜져야 하고, 선택 참석자의 조건은 참고 정보로 충분하다"는 판단에 설계를 걸었습니다. 이미 주어진 구분을 그대로 신뢰하면 추가 입력 없이, 후보 시간 리스트 한 화면의 정보 설계만으로 문제를 풀 수 있기 때문입니다. 회의실 예약이나 반복 회의 같은 부가 기능은 이 문제와 직결되지 않아 다루지 않았습니다.

## 프로토타입에서 눌러볼 것

프로토타입은 세 입장(주최자, 필수 참석자, 선택 참석자)으로 나뉘어 있습니다. 첫 화면에서 입장을 고르고, 상단 칩으로 언제든 다른 입장으로 건너뛸 수 있습니다.

1. **참석자 등록의 필수/선택 토글** (주최자 2/6). 정하늘·한유진 님의 참석 무게를 토글로 바꿔볼 수 있습니다. 이 구분 하나가 이후 모든 화면의 기준이 됩니다.
2. **후보 시간 리스트** (주최자 4/6). 모든 후보가 "필수 4/4 가능"을 만족한 상태에서, 회피 시간 겹침과 선택 참석 가능 인원이 신호로 붙어 있습니다. 카드를 누르면 확정 버튼이 활성화되고, 고른 카드의 정보가 확정 화면까지 이어집니다. 이 화면이 설계의 핵심입니다.
3. **빈 상태** (후보 시간 리스트 하단). "예시: 필수 참석자의 업무 일정만으로 겹치는 시간이 없다면?"을 누르면, 화면을 비워두는 대신 탐색 기간을 늘리거나 참석의 무게를 조정해 재탐색하는 흐름이 나옵니다.
4. **제약 입력 그리드** (필수·선택 참석자 2단계). 캘린더에서 가져온 업무 일정(주황)은 고정돼 있고, 빈 칸을 누르면 피하고 싶은 시간(빨강)을 직접 표시하고 지울 수 있습니다. 같은 그리드지만 범례 문구가 입장에 따라 다릅니다 — 필수 참석자에게는 "후보에서 제외돼요", 선택 참석자에게는 "우선 제안에 참고돼요"라고 안내합니다.
5. **선택 참석자의 건너뛰기** (초대·입력 화면). 두 화면 모두에서 건너뛸 수 있고, 응답하지 않아도 확정 통보까지 흐름이 그대로 이어집니다.
6. **유형별 확정 통보** (주최자 6/6, 각 참석자 마지막 화면). 필수 참석자에게는 "피하고 싶은 시간을 피해서 잡았다"는 확인을, 선택 참석자에게는 "참석이 어려우면 빠져도 괜찮다"는 메시지를 보냅니다.
