# 모두에게 괜찮은 시간, 어떻게 찾을까

> 6명이 모이는 1시간짜리 회의. 누군가는 점심 직후를 피하고 싶고, 누군가는 특정 요일에 외근이 많고, 꼭 와야 하는 사람과 빠져도 되는 사람이 섞여 있습니다. 이 조건들을 반영해 회의 시간을 빠르게 확정하는 경험을 설계했습니다.

## 어떤 문제인가

회의 일정을 잡을 때 6명의 조건을 모두 같은 무게로 다룰 수는 없습니다. 누군가는 점심 직후 한 시간만 피하면 되지만, 누군가는 특정 요일 전체가 빕니다. 그중에서도 가장 중요한 차이는 참석의 무게입니다. 꼭 참석해야 하는 사람이 있고, 빠져도 괜찮은 사람이 있습니다.

그런데 기존 일정 조율 도구에는 이 차이를 담을 수 없는 한계가 있습니다. 응답자 전원을 동일하게 취급해 **"전원 가능한 시간"만 찾다** 보니, 빠져도 괜찮은 한 명의 제약이 필수 참석자 전원이 가능한 현실적인 시간까지 지워버립니다. 결국 주최자는 도구 대신 메신저 개별 조율에 많은 시간을 쏟게 됩니다.

## 어떻게 풀었나

참석자를 **필수 / 선택**으로 구분해 받고, 이 구분을 탐색 로직과 화면 양쪽에 그대로 반영했습니다.

- **필수 참석자의 제약 = 하드 필터.** 회피 시간대·외근 요일을 어기는 시간은 후보에서 제외합니다.
- **선택 참석자의 제약 = 소프트 신호.** 후보를 거르지 않고 각 후보 옆에 "선택 N/M명 가능"으로만 표시합니다.
- **무응답이 전체를 막지 않습니다.** 선택 참석자가 응답하지 않아도 필수 참석자 응답만으로 탐색이 진행됩니다.
- **통보도 유형별로 다르게.** 필수 참석자에게는 확정 사실을, 선택 참석자에게는 "참석이 어려우면 빠져도 괜찮다"는 메시지를 전달합니다.

## 프로토타입에서 눌러볼 것

1. **참석자 등록 화면** — 필수/선택 토글로 참석자 유형을 바꿔보세요.
2. **후보 시간 리스트** — "필수 4/4 가능 + 선택 2/2 가능"처럼 후보마다 정보량이 붙어 있습니다. 이 화면이 설계의 핵심입니다.
3. **빈 상태** — 후보 리스트 하단의 *"예시: 후보 시간이 하나도 없다면?"* 을 누르면, 어떤 조건 때문에 막혔는지 보여주고 조건 하나를 완화해 재탐색하는 흐름을 볼 수 있습니다.
4. **확정 완료 화면** — 필수/선택 참석자에게 서로 다른 톤의 통보 메시지가 발송되는 것을 확인할 수 있습니다.
