# 모두에게 괜찮은 시간, 어떻게 찾을까

6명의 동료가 1시간 회의를 잡아야 할 때, 필수·선택 참석자를 구분해 빠르게 확정하는 경험을 설계했습니다.

## 바로 보기

- **[설계 과정 — 문제 정의부터 왜 이렇게 설계했는지](https://lgh0829.github.io/meeting-scheduler-case-study/design-process.html)**
  문제 정의, 핵심 설계 원칙, 플로우, 대안 비교까지 8장으로 정리한 케이스 스터디입니다. 방향키 또는 스와이프로 넘겨보세요.
- **[프로토타입 — 클릭 가능한 핵심 플로우](https://lgh0829.github.io/meeting-scheduler-case-study/index.html)**
  회의 만들기부터 확정 통보까지 7개 화면을 실제 클릭으로 체험할 수 있습니다.

## 핵심 아이디어

기존 일정 조율 도구는 참석자를 동일한 조건으로 취급해 "전원 만장일치 시간"만 찾아줍니다. 이 설계는 참석자를 필수/선택으로 구분해, 필수 참석자의 제약은 하드 필터로 적용하고 선택 참석자의 제약은 "N/M명 가능"이라는 정보로만 보여줍니다.
