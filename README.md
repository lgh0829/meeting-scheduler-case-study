# 모두에게 괜찮은 시간, 어떻게 찾을까

> 6명이 모이는 1시간짜리 회의. 누군가는 점심 직후를 피하고 싶고, 누군가는 특정 요일에 외근이 많고, 꼭 와야 하는 사람과 빠져도 되는 사람이 섞여 있습니다. 이 조건들을 반영해 회의 시간을 빠르게 확정하는 경험을 설계했습니다.

## 어떤 문제인가

회의 일정을 잡을 때 6명의 조건을 모두 같은 무게로 다룰 수는 없습니다. 가장 중요한 차이는 참석의 무게입니다. 꼭 참석해야 하는 사람이 있고, 빠져도 괜찮은 사람이 있습니다.

그런데 기존 일정 조율 도구에는 이 차이를 담을 수 없는 한계가 있습니다. 응답자 전원을 동일하게 취급해 **"전원 가능한 시간"만 찾다** 보니, 빠져도 괜찮은 한 명의 제약이 필수 참석자 전원이 가능한 현실적인 시간까지 지워버립니다. 결국 주최자는 스케줄링 도구 대신 메신저 도구를 사용하여 개별 조율에 많은 시간을 쏟게 됩니다.

## 어떻게 풀었나

주최자로부터 참석의 무게를 스케줄링 도구가 알게 했습니다. 여기에 제약의 성격(업무 일정 때문에 불가능한 시간인지, 피하고 싶은 시간인지)을 한 축 더 얹어, 네 가지 조합을 각각 다르게 다룹니다.

| | 필수 참석자 | 선택 참석자 |
|---|---|---|
| **업무 일정** (외근, 다른 회의) | **하드 필터.** 겹치는 시간은 후보에서 제외합니다 | **1순위 신호.** 참석 가능한 인원이 많을수록 후보가 위로 올라갑니다 |
| **선호 시간** (점심 직후 회피 등) | **1순위 신호.** 회피 시간을 피한 후보가 위로 올라갑니다 | **2순위 신호.** 후보가 비슷할 때 참고합니다 |

- **선택 참석자의 무응답은 진행을 막지 않습니다.** 필수 참석자의 응답만 모이면 후보 탐색이 시작됩니다.
- **확정 통보도 무게에 맞춥니다.** 필수 참석자에게는 확정 사실을, 선택 참석자에게는 "참석이 어려우면 빠져도 괜찮다"는 메시지를 보냅니다.

후보 시간은 "전원 가능한 시간" 하나가 아니라 어떤 조건을 얼마나 만족하는지 비교할 수 있는 정보로 나타나고, 마지막 선택은 주최자가 내립니다.

## 왜 이 방식으로 설계했나

### 가설과 검토한 대안

"필수 참석자의 조건은 반드시 지켜져야 하고, 선택 참석자의 조건은 참고 정보로 충분하다" — 이 한 문장의 가설에 설계를 걸었습니다. 배제한 대안은 두 가지입니다.

| 대안 | 내용 | 배제한 이유 |
|---|---|---|
| **A. 취합 자동화만 강화** | 흩어진 응답을 한곳에 모으는 데 집중 | 취합을 자동화해도 탐색 로직이 전원 만장일치를 요구하면 같은 문제가 반복됩니다. 취합은 이 설계의 전제 단계로 포함했습니다 |
| **B. 참석자별 가중치 점수화** | 1~5점 척도로 우선순위를 세분화 | 주최자에게 참석자마다 점수를 매기는 입력 부담을 더합니다. 필수/선택이라는 이분법이 이미 주어져 있어, 그보다 정교한 표현은 과한 설계라고 판단했습니다 |

이미 주어진 구분을 그대로 신뢰하면 추가 입력 없이 문제를 풀 수 있습니다. 회의실 예약이나 반복 회의 같은 부가 기능은 이 문제와 직결되지 않아 다루지 않았습니다.

### 가설이 화면에 반영된 방식

- **참석의 무게는 등록 단계에서 토글 한 번으로** 정합니다. 이후 응답 현황, 후보 리스트, 확정 통보까지 모든 화면이 이 구분을 기준으로 갈라집니다.
- **후보 시간은 "가능/불가"가 아니라 신호 묶음**(필수 4/4 가능 · 회피 겹침 N건 · 선택 N/M 가능)으로 나타납니다. 도구는 후보를 줄여주고, 마지막 판단 근거는 주최자에게 그대로 보여줍니다.
- **막다른 화면을 만들지 않았습니다.** 후보가 하나도 없으면 어떤 조건이 막았는지 원인을 보여주고, 탐색 기간 확대나 참석 무게 조정 같은 다음 행동을 바로 제안합니다. 선택 참석자가 응답하지 않아도 흐름은 끊기지 않습니다.

### 익숙한 패턴을 맥락에 맞게 다듬은 방식

- **주간 시간표 그리드**는 일정 조율 도구에서 익숙한 패턴이지만, 캘린더에서 자동으로 가져온 업무 일정(고정)과 직접 눌러 표시하는 회피 시간(토글)을 색과 인터랙션으로 분리해, "불가능한 시간"과 "피하고 싶은 시간"이 다르게 처리된다는 것을 화면 자체가 말하게 했습니다.
- **같은 화면도 입장에 따라 문구가 다릅니다.** 동일한 제약 입력 그리드라도 필수 참석자에게는 "이 시간은 후보에서 제외돼요", 선택 참석자에게는 "우선 제안에 참고돼요"라고 안내해, 내 입력이 어떻게 쓰이는지를 그 자리에서 알 수 있습니다.
- **필수/선택 구분은 색 체계로 끝까지 이어집니다.** 초대 배지, 응답 상태 칩, 확정 메시지의 톤까지 하나의 규칙으로 연결되고, 라이트/다크 모드 모두 대응합니다.
